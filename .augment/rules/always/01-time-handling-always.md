# 时间处理强制规则

## 🚨 强制执行声明
**本规则是强制性的，AI必须无条件遵循，不得以任何理由违反。**

## 📋 规则内容

### 🚫 绝对禁止的行为
- ❌ 使用任何硬编码时间（如"2025-01-08", "2024-12-25"等）
- ❌ 使用训练数据中的时间信息
- ❌ 假设当前时间或日期
- ❌ 使用相对时间描述而不验证实际时间

### ✅ 强制要求的行为
- ✅ 每次需要时间时必须调用系统API
- ✅ 使用 `date '+%Y-%m-%d %H:%M:%S'` 获取实际时间
- ✅ 在文档中使用实际系统时间
- ✅ 验证时间信息的准确性

## 🔧 强制执行机制

### 每次需要时间时必须执行
```bash
# 强制执行命令 - 不可省略
CURRENT_TIME=$(date '+%Y-%m-%d %H:%M:%S')
CURRENT_DATE=$(date '+%Y-%m-%d')
echo "当前时间: $CURRENT_TIME"
```

### 文档生成时间戳格式
```markdown
**创建时间**: $(date '+%Y-%m-%d')
**最后更新**: $(date '+%Y-%m-%d %H:%M:%S')
```

### 代码中的时间处理
```rust
// Rust示例 - 获取实际系统时间
use chrono::{DateTime, Utc};
let now: DateTime<Utc> = Utc::now();
println!("当前时间: {}", now.format("%Y-%m-%d %H:%M:%S"));
```

## 🔍 验证机制

### 自动验证命令
```bash
# 检查是否包含硬编码时间
grep -E "2025-01-|2024-12-|2023-" *.md && echo "❌ 发现硬编码时间，违反规则"

# 检查是否使用了实际系统时间
current_date=$(date '+%Y-%m-%d')
grep "$current_date" *.md && echo "✅ 使用了实际系统时间"
```

### 手动验证清单
- [ ] 检查所有时间戳是否为实际系统时间
- [ ] 确认没有硬编码日期
- [ ] 验证时间格式的一致性
- [ ] 确认时间信息的准确性

## 🚨 违规处理

### 发现违规时的处理流程
1. **立即停止** - 停止当前操作
2. **调用系统API** - 执行 `date` 命令获取实际时间
3. **替换硬编码时间** - 用实际时间替换所有硬编码时间
4. **重新验证** - 确认修正结果
5. **继续执行** - 修正完成后继续任务

### 违规示例和修正
```bash
# ❌ 错误示例
echo "创建时间: 2025-01-08"

# ✅ 正确修正
echo "创建时间: $(date '+%Y-%m-%d')"
```
